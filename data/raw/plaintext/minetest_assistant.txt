Everyone knows about the Minecraft video game. It allows people to create their own worlds by cubes, mining some resources from the ground and interacting with monsters called “mobs”.

But not so many people knows about its free and undemanding clone named Minetest. Actually, its engine is just a collection of cubes without any resources like furniture, mobs, etc. Some modifications fix this problem by the implementation of all necessary addons. The most known mod that implements the basic Minecraft functionality is MineClone 2.

Sometimes it is challenging to find any mobs. They can blend into the background like grass or be hidden in the night time. Fortunately, it can be solved by using Object Detection. One of the most popular models is Ultralytics YOLOv8, implemented in the PyTorch deep learning framework.

Detection Model
First of all, you need to mark up mobs by highlighting them with the “bounding box” and assigning them one of available classes. Computer Vision Annotation Tool can help with it. This project consists of 25 mob classes from all three game dimensions: The Overworld, The Nether and The End. The resulting dataset consists of 1,000 samples.

Then, you need to train a model by providing the given dataset. There are a lot of YOLOv8 versions. The YOLOv8-Nano model, the smallest one, is chosen due to performance limitations. This model is pretrained on the COCO dataset. As the cheap GPU hosting, vast.ai can be used. The training GPU is NVIDIA GeForce RTX 4090 (24 GB). To start training, this command is used:

yolo detect train data=minetest.yaml model=yolov8n.pt epochs=1000 batch=-1 patience=0 imgsz=640

The inference GPU is NVIDIA GeForce GTX 1050 Ti (4 GB). To validate the model, you can use this command:

yolo val model=yolov8n.pt data=minetest.yaml batch=32 plots=True device=0

Confusion Matrix (PyTorch)
The original model format is the PyTorch *.pt format. It can be accelerated by using the NVIDIA’s TensorRT technology with the *.engine format. To make conversion, run this command:

yolo export model=yolov8n.pt format=engine
To make comparison between these two formats, it is possible to run the validation:

yolo val model=yolov8n.engine data=minetest.yaml batch=32 plots=True device=0

In-Game Overlay Window
The YOLOv8 framework allows people to detect objects on different images and videos. But to use the model in the game, the program should be built.

This application is a Python in-game overlay window. It consists of three threads: Key Capture, Object Detection and UI Rendering.

Is is based on the PyQt5 GUI framework. To get screenshots from the screen, the mss library is used. To capture key clicks, the pynput package is used.

Minetest Assistant Scheme
The overlay window consists of three UI elements: the bounding boxes with the class-score information, thetext description of the found mobs and the number of frames per second. They can be turned off by the buttons defined in the settings.
